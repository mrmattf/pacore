# ADR-015: Assessment-First Sales Motion (Skills Diagnostic)

## Status
Accepted

## Context

### The Cold-Start Problem

Selling a $149/month Shopify automation app against an established app store requires: brand recognition, ratings and reviews, marketing budget, and a self-serve onboarding experience that converts browsers to active users without human intervention. Clarissi has none of these at launch.

Alternatives used in the market:
- **Free trial → paid conversion:** Works when the product demonstrates value within minutes of installation. Clarissi's skills require webhook configuration, credential setup, and real order volume before value is visible — too slow for free trial conversion.
- **Demo → close:** Generic demos require the prospect to imagine their own workflow; conversion is low without their actual data.
- **Consulting → retainer:** Generic consulting commoditizes the engagement; there's no platform moat.

### The Insight: The Assessment IS the Product

Clarissi's MCP tools can analyze a customer's live Shopify and Gorgias data and generate a concrete operational picture before any skill is activated. This is not generic consulting — it is the platform doing analysis before execution.

**What makes this differentiated:**
- No generic consultant can produce a ticket deflection projection backed by live API data
- No Shopify app can analyze the customer's data before installation
- The agent running the Assessment uses the exact same MCP infrastructure the customer will use in production — the Assessment IS the product demo
- Skill Gap Analysis identifies automation opportunities beyond the current catalog — gives Clarissi a direct read on the product roadmap from real demand

### Existing Infrastructure

The Assessment is already partially built. Existing MCP tools provide the core data layer:

| Tool | Data Returned | Assessment Use |
|------|--------------|----------------|
| `shopify_check_inventory` | Variant availability + backorder flag | Section 1: current backorder exposure |
| `shopify_get_inventory_eta` | Restock timeline per variant | Section 1: exposure severity |
| `shopify_get_order` | Order + customer context | Section 1: affected order count |
| `pacore_list_skill_templates` | Skill catalog with slot requirements | Section 2: skills match matrix |
| `pacore_get_execution_log` | Prior execution history | Section 2: existing automation baseline |

**What's missing:**
- `gorgias_get_tickets` — pull last 90 days of ticket volume by category (1 week to build)
- Skills Gap Analysis prompt — LLM reasoning over ticket categories that don't map to existing skills (1 week of prompt engineering)
- Report formatter — structured JSON → PDF (2 weeks; can be a manual template initially)

## Decision

**The primary top-of-funnel motion is a paid Skills Assessment ($1,500–2,500 one-time) powered by existing MCP tools. The Assessment produces an Automation Readiness Report covering existing skill matches AND a Skill Gap Analysis for capabilities not yet built.**

### The Automation Readiness Report

A four-section report delivered as PDF + 30-min readout call:

**Section 1 — Current Exposure**
- Backorder exposure: % of SKUs currently at negative inventory, # orders affected in last 30 days
- Inventory ETA summary: which backordered items have known restock dates vs. unknown
- Gorgias ticket category breakdown: volume by type (backorder, shipping exception, fraud, order status, etc.)
- Generated by: agent invoking `shopify_check_inventory`, `shopify_get_inventory_eta`, `gorgias_get_tickets`

**Section 2 — Skills Match Matrix**
- For each of the 4 current skills: estimated monthly fire rate based on customer data + expected outcome
- Example: "Backorder Notification: would have fired on ~180 orders last month. Estimated deflection: 120 Gorgias tickets/month at $5/ticket = $600/month labor saved."
- Generated by: agent cross-referencing `pacore_list_skill_templates` against ticket categories and order patterns

**Section 3 — Skill Gap Analysis**
- Ticket categories with no matching current skill and volume above 100/month are flagged as gaps
- Each gap gets: name, trigger description, estimated ticket deflection, estimated build timeline, priority ranking
- Example: "347 tickets/month tagged 'order_status_update' — no skill covers this. Candidate: Order Status Update Skill (Shopify → Gorgias). Estimated deflection: 200 tickets/month."
- Generated by: LLM reasoning over ticket category distribution vs. skill catalog

**Section 4 — ROI Projection**
- Existing skills activated: estimated ticket deflection × $5/ticket vs. Concierge retainer cost
- Payback period: months to break even on Assessment fee at projected deflection rate
- Gap skills: additional savings if gap skills were built (shown as "potential upside")

### Sales Motion

1. Prospect identified (Track 1 relationship, inbound, or outbound to ICP brands)
2. Offer Assessment as a no-risk first step: "We'll analyze your actual Shopify and Gorgias data and show you exactly what's possible — no commitment required"
3. Customer grants API credentials → operator configures agent with customer credentials (per-request credential pattern via HTTP headers, already implemented)
4. Agent runs against live data in 5–10 minutes → produces report draft
5. Operator reviews, adds commentary, assembles PDF
6. Delivered in 3–5 business days (time is now operator review, not data gathering)
7. 30-min readout call: walk through data, project ROI, give tier recommendation
8. Assessment fee credited toward first Concierge month if they convert to Concierge within 30 days

**Tier recommendation output (explicit, not implicit):**

| Customer Profile | Recommended Tier | Signal |
|-----------------|-----------------|--------|
| 500+ tickets/month, complex stack, wants managed outcomes | Concierge | Full ROI projection justifies retainer |
| Lower volume, simple stack, technical buyer | Self-Serve | ROI positive but retainer overhead not warranted yet |
| Skill gaps requiring new custom skills | Concierge | Only operator-led development fills gap skills |

The Automation Readiness Report concludes with an explicit recommendation: "We recommend Self-Serve" or "We recommend Concierge [Starter/Standard/Growth]" with reasoning. See ADR-016 for the full three-tier model.

**Target conversion rate:** 2/3 Assessments → Concierge retainers; remaining 1/3 directed to Self-Serve (still a paid outcome of the Assessment)

### Pricing

- $1,500–2,500 one-time (depends on customer stack complexity)
- Credited toward first Concierge month if they convert within 30 days
- At 2/3 conversion: net cost of Assessment per Concierge client = $500–850 (cost of acquisition vs. $2,500/month ACV → 0.2–0.3 month payback)

### Minimum Viable Test (No Engineering Required)

The Assessment can be run manually before `gorgias_get_tickets` is built:

1. Configure existing backorder MCP tools with prospect's Shopify credentials
2. Run agent in Claude Desktop → produces backorder exposure section automatically
3. Manually pull Gorgias ticket data via Gorgias API (curl or Postman)
4. Analyst adds commentary + gap analysis based on ticket category inspection
5. Deliver PDF assembled from a template

**Goal of manual test:** Validate that customers find the report credible and actionable, and that 2/3 convert to Concierge. The format is worth proving before engineering investment.

### Skill Gap Analysis as Product Roadmap Signal

Every Assessment Skill Gap Analysis is a demand signal for the platform roadmap. The gaps identified across 10+ Assessments reveal the highest-value, highest-volume skills to build next.

This creates a compounding advantage: the Assessment motion both closes Concierge deals AND generates the most reliable product roadmap signal available — direct customer demand backed by live API data, not surveys or interviews.

### Vertical Expansion

The Assessment motion is vertical-agnostic. For a new vertical:
1. Connect the vertical's key APIs as MCP tools (e.g., Clio for legal, NetSuite for manufacturing)
2. Map ticket/event volume against available skill catalog
3. Report format stays the same — Section 1 changes to the vertical's event types

The Assessment is the fastest path to validating a new vertical: one Assessment with real customer data is more signal than months of market research.

## Consequences

### Positive

- **Higher conversion than self-serve:** Customer sees their own data, not a generic demo — dramatically higher credibility
- **Paid top-of-funnel:** Assessment itself generates revenue ($1,500–2,500) while qualifying prospects
- **Platform roadmap demand signal:** Skill Gap Analysis across Assessments = most reliable product roadmap data available
- **Assessment IS the product demo:** The agent invoking MCP tools shows exactly how the platform works — no separate demo infrastructure needed

### Negative

- **Operator time per Assessment:** ~5–8 hours (setup, review, report assembly, readout call). At $50/hour operator cost = $250–400 per Assessment. Net at 2/3 conversion: ~$375–600 per converted client (excellent CAC).
- **Manual until `gorgias_get_tickets` is built:** Gorgias data must be pulled manually for the first 3–5 Assessments.
- **Requires prospect to grant API credentials:** Some prospects will hesitate. Mitigated by framing as read-only access, short-duration (Assessment only), and offering to use a test/read-only API key.

### Mitigation

- Build `gorgias_get_tickets` MCP tool as the first new tool after Assessment motion is validated (P3 in ADR-014)
- Create a credential-request template that explains exactly what access is needed and why (read-only, specific scopes, not stored after Assessment unless they convert)
- First 3 Assessments are the validation test — if conversion rate is below 50%, revisit report format before investing in engineering
